# InstantMesh 使い方ガイド（フェーズ1・開発版）

このドキュメントは、現在の実装（`cmd/server` / `cmd/client`）を **実際に起動して動かす**ための手順です。プロダクトとしての UX フロー（GitHub/Google ログイン等）は [`README.md`](../README.md) の「使い方（UXフロー）」を、要件・設計は [`docs/要件定義書.md`](要件定義書.md) / [`docs/システムアーキテクチャ定義書.md`](システムアーキテクチャ定義書.md) を参照してください。

> **前提**
> - Go が必要です（開発環境では `go run` で直接実行できます）。
> - 仮想NIC を実際に張る `-tunnel` は **既定で有効**で、**管理者/root 権限**が必要です（Windows は管理者 PowerShell、Linux/macOS は `sudo`）。権限なしでシグナリングの一連フロー（ルーム作成 → 参加申請 → 承認 → エンドポイント交換）だけを確認したいときは **`-tunnel=false`** を付けます（この場合は仮想NIC を張らず実通信はしません）。

---

## 1. サーバーは何を担うか

`cmd/server` は 1 バイナリで **2 つの役割**を単一プロセスで提供します（`buildServer`）。役割はコード上 `pkg/hub`（制御）と `pkg/relayhub`（データ）に論理分離されています。

| エンドポイント | 役割 | 中身 | DERP 相当か |
| :-- | :-- | :-- | :-- |
| `/ws`（WebSocket） | **シグナリング（コントロールプレーン）** | ルーム作成・待合室承認・キック・エフェメラル管理・STUN で発見した WAN エンドポイント（`peer_info`）の交換 | いいえ |
| `/relay`（WebSocket） | **リレー（データプレーン）** | P2P 直通が失敗したときだけ、暗号化パケットを宛先公開鍵ベースで**素通し転送** | **はい（DERP 相当）** |

- 「DERP サーバー」なのは `/relay` の部分だけで、本体はまずシグナリングサーバーです。
- **サーバーは通信内容を復号しません。** リレーも E2E 暗号化されたまま宛先公開鍵で転送するだけです（設計原則2）。扱うのは公開鍵とメタデータのみ。
- リレーは常時経路ではなく、**直通失敗時のフォールバック**です（`pkg/connmon` が直通ハンドシェイクの成立を検知できなければ転落）。無料プランは 1 接続 100MB 超で 64kbps に速度制限。

### 起動

```bash
go run ./cmd/server -addr :8080
```

| フラグ | 既定 | 意味 |
| :-- | :-- | :-- |
| `-addr` | `:8080` | 待ち受けアドレス |
| `-path` | `/ws` | シグナリング（制御プレーン）のパス |
| `-relay-path` | `/relay` | リレー（データプレーン）のパス |
| `-pool` | `10.0.0.0/8` | ルームへ払い出す /24 の元ブロック（`<= /24`） |
| `-maintenance-interval` | `30s` | ルーム掃除・待合室失効の周期 |
| `-shutdown-grace` | `5s` | グレースフルシャットダウンのタイムアウト |

## 2. サーバーの置き場所（public である必要はあるか）

**AWS である必要はありません。** 本質的な要件は次の 1 点だけです。

> シグナリング（`/ws`）とリレー（`/relay`）は、**ホストとゲスト全員から到達できるアドレス**に置く必要がある。

ホストもゲストも、招待リンクに書かれた `server`（例 `ws://host:8080/ws`）へ**自分から発信して**接続します。全員がそこへ繋がって初めてルームが成立するため、置き場所は利用シーン次第です。

| 利用シーン | サーバーの置き場所 | public 必要か |
| :-- | :-- | :-- |
| 全員が同じ LAN / VPN 内 | LAN 内の 1 台（例 `ws://192.168.x.x:8080/ws`） | **不要** |
| 別ネットワーク越し（社外テスター・リモート開発） | インターネットから到達可能なホスト | **必要**（AWS 限定ではない） |

「インターネットから到達可能」の実現手段は AWS に限らず、VPS・自宅サーバー＋ポート開放/DDNS・cloudflared 等のトンネルでも成立します。`docs/システムアーキテクチャ定義書.md` が AWS（EC2 Graviton・`ap-northeast-1`）を挙げているのは**本番展開の想定**であり、開発版の動作要件ではありません。

なお public に置いても中央集権 VPN とは性質が異なります: ①直通成功時はサーバーを一切経由せず、リレー時も暗号化のまま素通し ②鍵照合は招待リンク（帯域外）で行うためサーバー侵害時も盗聴不可 ③使い終われば消える（エフェメラル）。

## 3. クライアント（CLI / ヘッドレス）

`-mode` に `host` / `guest` / `gui` を指定します（**既定は `gui`**。省略すると GUI モードで起動します）。

### ホスト（ルームを作る人）

既定では公開サーバー（`wss://s1.instantmesh.net/ws`）へ **Cognito 認証**で接続します。ホスト起動時にブラウザで Cognito Hosted UI が開くので、サインイン（アカウントは**招待制**。管理者が作成）すると ID トークンを取得して接続します。

```bash
# 既定（公開サーバー ＋ Cognito 認証 ＋ -tunnel 有効）: フラグなしでホストできる（-tunnel が既定のため要管理者権限）
go run ./cmd/client -mode host

# ローカルの開発サーバー（DevAuthenticator）に繋ぐ場合は Cognito を無効化し -account を使う
go run ./cmd/client -mode host \
  -server ws://localhost:8080/ws \
  -cognito-domain= \
  -account dev-account \
  -duration 3600
# -tunnel/-stun とも既定で有効（要管理者権限）。フラグなしで穴あけまで試せる。
# 権限なしでシグナリングのみ確認するなら -tunnel=false を付ける（STUN も止めるなら -stun= を併記）。
```

> **`-cognito-domain=`（空文字）で Cognito を無効化**します。既定では `-cognito-domain` / `-cognito-client-id` に公開サーバーのユーザープール値が入っており、ホスト時に PKCE サインインが走ります。ローカルの `DevAuthenticator` サーバー相手ではこれを空にして `-account` を Bearer に使います。`-cognito-redirect` は Cognito 登録値に固定のためフラグ廃止（`http://localhost:53682/callback`）。

- 起動すると標準出力に **招待リンク**（`instantmesh://join?...`）・**SAS**（帯域外照合用の短縮フィンガープリント）・**QR コード**が表示されます。これを**安全な経路**（対面・別チャネル）でゲストへ渡します。
- **既定は手動承認**（`-auto-approve=false`）。ゲストの参加申請が届くと標準出力に `nick` と `SAS` が表示されるので、**SAS を帯域外で照合**したうえで標準入力からコマンドで承認/拒否します（デモで自動承認に戻すなら `-auto-approve=true`）。
- 稼働中に標準入力へ打てるコマンド（先頭数文字での指定可・大小無視）:

  | コマンド | 別名 | 動作 |
  | :-- | :-- | :-- |
  | `approve <公開鍵先頭>` | `a` | 待合室のゲストを**承認** |
  | `reject <公開鍵先頭>` | `deny` | 待合室のゲストを**拒否** |
  | `list` | `l` | 待合室の申請一覧を表示 |
  | `rotate` | `r` / `reissue` | **招待リンクを再発行**（旧リンクは即失効・承認済みメンバーは維持） |
  | `help` | `h` / `?` | コマンド一覧を表示 |

  申請が 1 件だけなら `approve`（引数なし）でその 1 件を承認できます。GUI モードではブラウザの待合室ボタンで承認/拒否します。

### ゲスト（参加する人）

```bash
# -tunnel/-stun とも既定で有効のため要管理者権限（権限なしでシグナリングのみ確認するなら -tunnel=false）
go run ./cmd/client -mode guest \
  -invite "instantmesh://join?..." \
  -nick alice
```

- 招待リンクを解析 → 参加申請 → ホストの承認後に割当 IP（例 `10.0.0.x`）でメッシュ通信を開始します。
- 承認時、リンクに埋め込まれたホスト公開鍵と照合し、不一致なら **MITM 検知で中止**します（帯域外検証）。

### クライアントの主なフラグ

| フラグ | 既定 | 対象 | 意味 |
| :-- | :-- | :-- | :-- |
| `-mode` | `gui` | 共通 | `host` / `guest` / `gui`（省略時は `gui`） |
| `-server` | `wss://s1.instantmesh.net/ws` | host / gui | シグナリング URL（既定は公開サーバー。ローカル検証時は `ws://localhost:8080/ws` を指定） |
| `-account` | `dev-account` | host / gui | ローカル `DevAuthenticator` サーバー用の Bearer。**Cognito 有効時（既定）は無視される** |
| `-cognito-domain` | 公開プールの Hosted UI | host / gui | Cognito Hosted UI ベース URL。**空文字（`-cognito-domain=`）で Cognito を無効化**しローカル `DevAuthenticator` へ |
| `-cognito-client-id` | 公開アプリクライアント | host / gui | Cognito アプリクライアント ID（公開クライアント・シークレット無し） |
| `-cognito-scope` | `openid` | host / gui | 要求スコープ（カンマ区切り） |
| `-duration` | `3600` | host / gui | ルーム制限時間（秒） |
| `-auto-approve` | `false` | host | 参加申請を自動承認（デモ用）。既定は手動承認（標準入力に `approve`/`reject`） |
| `-invite` | `""` | guest | 招待リンク |
| `-nick` | `guest` | guest | ニックネーム |
| `-tunnel` | `true` | 共通 | wireguard-go 仮想NIC を起動（**既定で有効・要管理者権限**）。無効化は `-tunnel=false` |
| `-stun` | `stun.l.google.com:19302` | 共通 | STUN サーバー `host:port`（既定で WAN を発見し `peer_info` 広告）。無効化は `-stun=` |
| `-relay` | `true` | 共通 | 直通失敗時にリレーへ自動フォールバック（要 `-tunnel`） |
| `-ifname` | `wg-mesh` | 共通 | 仮想NIC 名（macOS は utun） |
| `-gui-addr` | `127.0.0.1:8088` | gui | GUI の待受アドレス（外部公開しない） |

## 4. クライアント（GUI / アプリ内ウィンドウ・ブラウザ）

`-mode gui`（**既定なので省略可**）で localhost に UI サーバーを立て、GUI から操作します。表示経路は OS で分岐します。

- **Windows**: OS 内蔵の WebView2 を使い、**アプリ内ウィンドウとして自動で開きます**（WebView2 ランタイム未導入などで失敗した場合のみ、既定ブラウザにフォールバック）。
- **Linux**: Chromium 系ブラウザ（chromium / google-chrome / microsoft-edge など）を `--app` モードで起動し、**アプリ内ウィンドウとして自動で開きます**（Chromium 系が見つからない場合は既定ブラウザにフォールバック）。
- **その他 OS（macOS 等）**: 既定ブラウザが**自動で開きます**。

いずれの経路でも待受 URL は `http://127.0.0.1:8088`（`-gui-addr`）で、手動でブラウザから同 URL を開いても操作できます。

`-mode`（既定 `gui`）も `-server`（既定 `wss://s1.instantmesh.net/ws`）も省略できるため、**フラグなしでそのまま起動**すれば公開サーバーに繋いだ GUI が立ち上がります。

```bash
# 既定（GUI ＋ 公開サーバー）でそのまま起動
go run ./cmd/client

# ローカルで立てたサーバー（DevAuthenticator）に繋ぐ場合は URL を明示し Cognito を無効化する
go run ./cmd/client -mode gui \
  -server ws://localhost:8080/ws \
  -cognito-domain= \
  -gui-addr 127.0.0.1:8088
```

- 起動後、自動で開くウィンドウ（Windows/Linux のアプリ内ウィンドウ、または既定ブラウザ）に **http://127.0.0.1:8088** が表示され、ルーム作成 / 参加 / **承認 / 拒否** / 招待リンク再発行 / 退出をボタン操作できます（CLI の標準入力コマンドと同じ待合室**手動承認**を、ボタン UI で行えます）。
- **既定で `-tunnel`/`-stun` が有効**です。GUI で「ルーム作成」や「参加」を行うと仮想NIC を起動するため、**管理者権限で起動**しておく必要があります（権限なしでシグナリングのみ試すなら `-tunnel=false` を付けて起動）。
- 待受は既定で `127.0.0.1` のみ（**外部公開しない**）。CSRF/DNS リバインディング対策で Origin/Host を検証し、状態変更は POST + 同一オリジンに限定。トークン等の秘密情報は UI へ配信しません（招待リンク・SAS など表示に必要なものだけ配信）。

## 5. ローカルで試す最短手順（3 ターミナル）

```bash
# ① サーバー
go run ./cmd/server -addr :8080

# ② ホスト（招待リンク / SAS / QR が出力される。ローカルサーバーは Cognito を無効化。権限不要でシグナリングのみ確認するため -tunnel=false）
go run ./cmd/client -mode host -server ws://localhost:8080/ws -cognito-domain= -account dev-account -tunnel=false

# ③ ゲスト（② で出た招待リンクを貼る）
go run ./cmd/client -mode guest -invite "instantmesh://join?..." -nick alice -tunnel=false
```

`-tunnel=false` を付ければ、**シグナリングの一連フロー**（ルーム作成 → 申請 → 承認 → `peer_info` 交換）を権限不要で確認できます。既定は `-tunnel` 有効なので、実際に仮想NIC を張って IP 疎通まで見るには両クライアントを**管理者権限**で実行し、STUN も使うなら `-stun ...` を付けます。実 NAT 越えでの穴あけ確立 / リレー切替の疎通検証は残タスクです（[`TODO.md`](../TODO.md) 参照）。

## 7. うまくいかないとき（トラブルシューティング）

### ゲスト参加時に「invalid or expired room token」

サーバーがそのトークンに対応する**生きているルーム**を見つけられないときに出ます。トークンの形式自体は正しく届いているので、原因は次のいずれかです。

- **古い招待リンクを使っている（最も多い）**: ホストを再起動すると毎回**新しいトークン**でルームが作り直されます（`create_room` ごとに新規発行）。前回の実行や再ビルド前に出たリンクは即無効です。→ **いま起動しているホストが出力した最新の招待リンク**を使ってください。
- **サーバー URL のズレ**: ホストの `-server` が `ws://localhost:...` だと招待リンクに `localhost` が埋め込まれ、別マシンのゲストは自分自身へ繋ぎに行きます。2台以上の構成では **LAN で届く IP**（例 `ws://192.168.x.x:8080/ws`）を指定してください。
- **ホスト切断でルーム解散**: エフェメラル設計のため、ホスト接続が切れるとルームは即解散します。ゲスト参加時にホストプロセスが起動・接続したままか確認してください。
- **招待リンクを再発行した**: 稼働中に `rotate` を打つと旧リンクは即失効します。新しいリンクを配ってください。

> 招待リンクは `&` などシェルの特殊文字を含みます。`-invite "instantmesh://join?..."` のように**必ずクォートで囲って**ください（囲わないとリンクが途中で切れ、別のエラーになります）。
